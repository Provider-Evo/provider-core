# WebUI 详细规则

## 一、目录与职责

### 主实现目录
- `src/webui/`

规则：
- 新功能和主页面实现应优先放到 `src/webui/`
- `src/webui/` 保持为内置管理台的唯一主实现位置

## 二、页面目标

WebUI 不是演示页，而是内置管理台（根路径 `/` 直接访问），至少要覆盖：
- 运行概览
- 平台状态
- 模型清单
- 配置摘要
- 运行日志反馈

## 三、页面交互要求

必须具备：
- 主题切换
- 便携设置
- 状态刷新
- 模型刷新
- 接口失败降级

推荐具备：
- 标签页切换
- 搜索过滤
- toast 提示
- 自动刷新控制
- 安全配置摘要展示

## 四、数据接口要求

为了减少前端拼装复杂度，应提供摘要接口，例如：
- `/v1/webui/summary`

摘要接口应尽量一次返回：
- 服务名
- 时间戳
- 安全配置摘要
- 平台状态
- 模型列表
- 能力矩阵
- 汇总计数

## 五、生产就绪要求

- 不依赖 CDN、外链字体、远程图标库
- 所有资源本地化或内联
- 任一接口失败时整页不得崩溃
- 要求在应用实际启动后可访问验证
- 页面模板过大时必须拆分为子模块，不应长期维持不可维护的超大单文件

## 六、当前项目实现要求

当前项目中至少应保持：
- `src/webui/routes.py`
- `src/webui/routers/`
- `src/webui/services/`
- `src/core/runtime_view.py`

这样可以把：
- 数据聚合
- 服务层
- 路由注册
- 页面入口
分离开，避免继续退化回简陋页面。
