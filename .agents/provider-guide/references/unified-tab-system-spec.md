# 终端与文件管理器选项卡统一设计与交互规范

## 总述
1. 终端、文件管理器共用同一套标签交互体系与布局规则，支持横向顶部、竖向侧边两种全局布局，竖向侧边均包含展开、压缩双形态，可全局统一切换，也可模块独立配置。
2. 两类标签可在同一标签栏内混合排列、统一管理，支持模块间一键联动跳转；终端会话与文件浏览/编辑会话相互独立，状态互不干扰。
3. 全模块遵循统一的懒加载与资源管控原则：非激活标签降频渲染、按需加载，海量内容分片缓存，避免多标签同时满载运行导致界面卡顿与内存溢出。
4. 全局统一交互范式：标签固定、拖拽排序、右键菜单、状态角标、撤销关闭、快捷键体系保持一致，降低跨模块学习成本。
5. **前后端生命周期分离规则**：仅前端WebUI关闭、网络断开、程序异常退出时，终端进程与后台任务持续保活；用户主动点击标签关闭按钮（X）为显式终止操作，确认后将彻底杀死对应终端进程、销毁会话，不做后台保留。文件标签主动关闭同理，直接移除会话，未保存内容触发二次确认。

## 一、终端选项卡设计与交互规范

### 1.1 布局模式
支持横向顶部、竖向侧边两种标签栏布局，可在设置中一键切换；竖向侧边标签栏包含展开、压缩两种显示形态，通过栏顶控件切换。所有布局切换均不改变标签排序、会话状态、输入缓冲区与输出历史，不中断正在运行的进程。

### 1.2 通用核心能力
1. 单标签对应一个独立终端会话，绑定Shell环境、工作目录、输出历史、滚动位置、输入状态与连接配置，切换标签时会话数据完整保留，后台运行进程不受影响。
2. 同一时间仅一个标签处于激活态，对应终端画布获得输入焦点；非激活标签会话后台保活，暂停画面渲染以节省系统资源。
3. 支持标签拖拽自由调整排序，固定标签始终前置排列；支持拖拽标签移出窗口生成独立终端窗口，拖回窗口则合并为标签。
4. 支持标签固定：常用会话可固定，固定标签前置排列且默认隐藏关闭按钮，防止误关闭。
5. 标签实时同步会话全量状态：启动中、连接中、在线运行、空闲、断开、异常退出、录制中、有新输出、离线保活中。
6. 支持会话记忆：程序重启后可一键恢复上次关闭的全部标签，本地终端恢复对应工作目录，远程终端保留主机配置并按配置决定是否自动重连。
7. 全链路资源管控：非激活标签暂停终端画面渲染，仅保留输出缓冲区；超长输出历史采用分片懒加载，滚动可视区域时动态加载对应历史片段，避免全量日志加载导致界面卡死。
8. 支持跨标签文本复制粘贴，支持批量向多个选中会话发送同一命令；支持全局搜索所有会话的输出历史。
9. 与文件管理器模块深度联动：文件管理器指定目录可一键打开对应路径的终端标签；终端内文件路径可点击唤起文件预览/编辑。
10. **进程保活边界**：仅前端页面关闭、网络异常断开、主程序异常崩溃时，后端保留终端进程持续运行；用户主动触发关闭操作（点击X、右键关闭、快捷键关闭）均为显式终止，直接销毁进程与会话。

### 1.3 横向顶部标签栏交互
1. 标签从左到右横向排列，单标签由状态指示点、终端类型图标、会话名称、关闭按钮组成。
2. 会话命名规则：本地终端默认命名「本地终端」，多开自动追加序号；远程终端默认使用主机备注/IP命名，支持双击重命名。名称过长自动截断，悬停展示完整名称、主机地址、工作目录与运行时长。
3. 激活态标签通过背景样式与主终端画布视觉衔接，关闭按钮常驻显示；非激活标签关闭按钮默认隐藏，鼠标悬停时浮现。
4. **关闭按钮行为**：点击关闭按钮（X），若会话无运行中进程，立即杀死终端进程、销毁会话、移除标签；若检测到存在运行中的前台进程、后台任务，弹出二次确认弹窗，确认后强制终止进程并关闭标签。
5. 标签数量超出容器宽度时，支持左右滚动浏览，尾部溢出标签收纳至下拉菜单，点击可快速跳转。
6. 标签栏最右侧固定新建按钮，点击展开下拉选项：快速新建本地终端、新建远程终端、从已保存主机列表选择。
7. 点击非激活标签立即切换会话，终端画布同步更新并自动获得输入焦点；非激活标签存在新输出时，标签展示未读角标与轻微闪烁提醒。
8. 标签栏右侧常驻全局功能入口：设置、全屏、录像审计、常用命令面板。
9. 鼠标中键点击标签触发快速关闭，规则与点击X一致；存在运行中进程时同样触发二次确认。

### 1.4 竖向侧边标签栏交互

#### 1.4.1 形态切换控件
位于竖向标签栏顶部，点击可在展开形态与压缩形态间平滑切换，切换过程中标签顺序、激活状态、固定状态、会话运行状态均保持不变。

#### 1.4.2 展开形态
1. 标签纵向依次排列，单标签横向包含「状态指示点 + 终端图标 + 会话名称 + 关闭按钮」。
2. 激活态标签通过背景高亮区分，关闭按钮常驻显示；非激活标签关闭按钮悬停浮现。
3. **关闭按钮行为**：点击关闭按钮（X），无运行进程则立即终止进程、移除标签；有运行进程则弹出二次确认，确认后强制杀死进程并关闭会话。
4. 点击非激活标签立即切换对应会话，主终端画布同步更新并激活输入焦点。
5. 固定标签置顶排列，附带固定标识；存在新输出的非激活标签展示未读角标。
6. 标签名称过长自动截断，悬停浮层展示完整会话信息：主机地址、工作目录、连接时长、当前运行进程。
7. 标签总数超出栏高时，支持鼠标滚轮纵向滚动浏览；支持拖拽调整标签排序。
8. 栏底固定新建按钮，点击展开创建选项：新建本地终端、新建远程终端。
9. 支持拖拽文件至标签区域，悬停延时后自动切换对应会话并自动填入文件绝对路径。

#### 1.4.3 压缩形态
1. 标签纵向等距排列，仅展示终端类型图标，标签栏收窄至适配图标宽度，最大化终端显示区域。
2. 悬停交互规则：
   - 悬停**当前激活态标签**：图标替换为关闭（X）图标，点击即触发关闭逻辑：无运行进程直接杀死进程并移除标签；有运行进程弹出二次确认，确认后终止进程并关闭。
   - 悬停**非激活态标签**：图标保持原样不变，点击则切换至对应终端会话。
3. 图标角标承载多类状态标识：连接中动画、断开/异常标记、固定标识、未读输出数量、录制中标记、离线保活标记。
4. 标签总数超出栏高时，支持鼠标滚轮纵向滚动浏览。
5. 栏底固定新建按钮，点击触发终端创建选项菜单。
6. 支持拖拽文件至图标标签，悬停延时后自动切换对应会话并填入路径。

### 1.5 终端类型与会话生命周期

#### 1.5.1 本地终端
1. **启动逻辑**：一键创建，无需额外配置，默认继承程序运行权限与系统Shell环境；默认工作目录可配置（程序目录、用户目录、上次路径）。
2. **多开规则**：支持无限多开，新建时按序号自动命名；支持复制当前会话，新会话继承原会话的工作目录与环境变量。
3. **状态流转**：
   - 启动中：展示加载动画，终端区域输出启动日志。
   - 运行中：状态点常亮，可正常输入输出。
   - 空闲：超过配置时长无操作，状态点变暗，进程保活。
   - 离线保活：前端WebUI关闭、网络断开后，进程持续运行，状态标记为离线保活，重连后自动同步输出；**主动点击关闭按钮不进入此状态，直接销毁进程**。
   - 进程退出：终端输出退出码，标签状态变红，支持一键重启恢复会话。
   - 异常崩溃：标签展示异常标识，提示崩溃原因，支持重启。
4. 支持切换Shell类型（bash/zsh/powershell等），切换后新标签生效。

#### 1.5.2 远程终端
1. **配置体系**：支持SSH协议连接，配置项包含主机IP/域名、端口、登录账号、验证方式（密码/私钥/二次认证）、跳板机代理、字符编码、超时时间。支持快速填充格式 `[账号:密码@]服务器IP[:端口]` 一键解析。
2. **连接流程**：
   - 发起连接：标签进入连接中状态，终端区域输出握手、认证日志。
   - 认证成功：状态切换为在线，自动进入用户默认目录。
   - 连接失败：标签标记异常，终端输出失败原因（网络不通、认证失败、端口不可达等），支持重试与编辑配置。
3. **保活与断开规则**：
   - 内置心跳保活机制，维持连接稳定。
   - 前端WebUI断开不触发SSH连接断开，后端持续维持连接，前端重连后直接挂载已有会话。
   - **主动关闭标签**：立即断开SSH连接、销毁会话实例、移除标签；有运行进程时二次确认，确认后强制断开连接。
   - 网络异常断开后，自动按配置间隔重试重连，重连过程标签展示重试状态与次数；超过最大重试次数后标记为断开，支持手动重连。
   - 重连成功后可按配置恢复会话上下文。
4. **配置管理**：主机配置可保存至全局主机列表，支持后续一键快速新建会话；标签内编辑配置仅当前会话生效，不影响全局保存的配置。

### 1.6 新建与会话管理交互
1. **快速新建**：点击新建按钮选择类型，立即创建对应终端标签并自动激活。
2. **从主机列表新建**：选择已保存的主机配置，一键创建远程终端并自动发起连接。
3. **复制会话**：右键选择「复制会话」，新建同配置终端标签；远程会话复用主机配置，本地会话继承工作目录。
4. **撤销关闭**：仅支持撤销短时间内主动关闭的标签，撤销后重新启动对应终端进程、恢复输出历史；超过保留时长的已关闭会话彻底销毁，不可恢复。
5. **批量操作**：支持框选多个标签，执行批量关闭、批量断开、批量重连、批量发送命令操作；批量关闭时统一检测运行中进程，批量确认后统一终止进程。

### 1.8 标签状态规则
- **启动/连接中**：标签展示加载动画，状态点闪烁，终端区域输出启动/连接日志。
- **在线运行中**：状态点常亮，标签正常显示，会话可正常交互。
- **空闲状态**：超过设定时长无输入输出，状态点变暗，会话后台保活。
- **离线保活中**：前端WebUI关闭、网络异常断开时，进程持续后台运行，标签标记保活状态；重连后自动恢复为在线状态。**主动关闭的标签不进入此状态**。
- **有新输出**：非激活标签、离线保活标签展示未读数量角标，伴随轻微视觉提醒，避免遗漏后台任务输出。
- **录制中**：终端操作录像开启时，标签展示录制标识，全局录像审计面板可统一管理。
- **断开/退出**：状态点高亮警示，会话名称置灰，终端区域展示退出码/断开原因。
- **异常状态**：认证失败、网络错误、进程崩溃等场景，标签展示异常标识，悬停提示具体错误原因。
- **后台任务运行中**：检测到长时间运行的进程时，标签展示运行中标识，关闭时触发二次确认。

### 1.9 快捷键体系

#### 标签管理
- Ctrl + T：新建默认终端
- Ctrl + Shift + T：撤销关闭最近标签（重新启动进程）
- Ctrl + W：关闭当前标签（终止对应终端进程）
- Ctrl + Tab / Ctrl + PageDown：切换到下一个标签
- Ctrl + Shift + Tab / Ctrl + PageUp：切换到上一个标签
- Alt + 1~9：快速切换到对应序号的标签
- Ctrl + Shift + D：复制当前会话到新标签

#### 终端操作
- Ctrl + R：重连远程终端 / 重启本地终端
- Ctrl + L：清空当前终端输出
- Ctrl + Shift + F：在当前会话输出中搜索
- F11：终端全屏显示
- Ctrl + Shift + C/V：终端内复制/粘贴

### 1.10 资源与性能管控
1. **渲染懒加载**：非激活标签暂停终端画面渲染，仅后台缓存输出内容；切换标签时仅渲染可视区域，超长历史滚动时分片加载。
2. **输出分片存储**：单会话输出超过阈值后自动分片，历史回溯时按需加载对应分片，避免单会话海量输出占用大量内存。
3. **资源自动回收**：主动关闭标签后，立即终止进程、释放系统资源，输出日志按配置保留时长供撤销恢复；超时后彻底清理。
4. **异常保护**：单会话异常崩溃不影响其他标签会话；批量关闭时按顺序释放资源，避免界面阻塞。
5. **空闲降频**：长时间未激活的标签自动降低输出刷新频率，进一步降低CPU占用。

### 1.11 会话持久化与进程保活机制
1. **前后端解耦架构**：终端Shell进程由后端服务独立托管，前端WebUI仅作为视图展示与输入入口。**关闭浏览器标签页、退出WebUI、前端网络断开，均不会终止后端终端进程**，运行中的脚本、任务、命令持续执行。
2. **主动关闭即终止**：用户通过标签X按钮、右键菜单、快捷键主动关闭标签，属于显式终止指令，立即执行：杀死本地终端进程 / 断开远程终端连接、销毁会话实例、清理运行时资源，仅按配置保留日志缓存用于短时间撤销恢复。
3. **会话唯一标识与PID绑定**：每个终端会话分配全局唯一会话ID，与Shell进程PID、工作目录、环境变量、输出缓冲区一一绑定，信息持久化落盘存储。主程序重启时，自动扫描系统中由本程序创建的存活终端进程，通过会话ID匹配对应会话数据。
4. **重启自动恢复挂载**：
   - 主程序异常中断后重新启动，自动扫描匹配存活的终端进程，原样还原所有对应标签、排序、固定状态与输出历史。
   - 进程仍在运行的会话，直接挂载接续，可继续输入操作，无任何中断感；进程已退出的会话，保留完整输出日志，标记为已退出状态，支持一键重启。
   - 恢复过程采用懒加载策略，优先还原激活会话的输出内容，非活跃会话历史后台异步加载，不阻塞启动速度。
   - **用户主动正常关闭的会话，重启后不自动恢复**，仅可在历史记录中查看日志。
5. **离线输出缓存同步**：前端断开、程序重启期间，终端所有输出内容持续写入持久化缓存。重新连接、重启恢复后，自动补全全部离线期间的输出日志，确保操作记录、任务日志无遗漏。
6. **会话保活策略配置**：
   - 可配置全局保活规则：永久保活、空闲指定时长后自动销毁、退出WebUI后保留指定时长。
   - 支持单会话单独设置保活等级，重要任务会话可锁定永久保活，禁止自动回收。
   - 保活规则仅对前端异常断开、程序异常退出生效，不影响用户主动关闭行为。
7. **多端同步挂载**：同一环境下多端打开WebUI，可同时挂载同一个终端会话，输入与输出实时同步；任意一端关闭页面，不影响会话运行与其他端的使用。
8. **异常兜底机制**：
   - 进程意外挂死、PID失效时，自动标记会话异常，保留最后输出日志，提供重启会话选项。
   - 主程序强制退出前，自动落盘所有会话的状态与输出快照，确保重启后最大程度还原。

## 二、文件管理器选项卡设计与交互规范

### 2.1 布局模式
与终端标签共用同一套布局体系，支持横向顶部、竖向侧边两种布局，竖向侧边包含展开、压缩双形态。与终端标签混合排列时，文件标签通过图标与终端标签区分，交互逻辑完全统一。布局切换不影响已有标签的目录状态、浏览历史、视图配置、编辑会话与预览状态。

### 2.2 通用核心能力
1. 单标签对应一个独立文件浏览会话，绑定当前目录路径、浏览历史记录、视图模式（图标/列表/详情）、排序筛选规则、文件选中状态、预览上下文与编辑会话。
2. 同一时间仅一个标签处于激活态，主浏览面板、地址栏、导航侧边栏同步联动更新为对应标签的内容与状态。
3. 支持标签拖拽自由调整排序，支持与终端标签混排；支持拖拽标签移出窗口生成独立文件管理器窗口。
4. 支持标签固定：常用目录标签可固定，固定标签前置排列，默认隐藏关闭按钮。
5. 标签实时同步会话状态：正常、加载中、路径无效、权限不足、后台任务进行中、编辑未保存、预览中。
6. 支持跨标签文件操作：拖拽文件/文件夹悬停至目标标签，延时自动切换至对应目录，释放后完成移动/复制操作。
7. 标签名称与当前目录名自动同步，路径变更时标签名实时更新；名称过长自动截断，悬停展示完整绝对路径。
8. 支持会话记忆，程序重启后可恢复上次打开的全部标签、对应路径与未保存的编辑会话。
9. 全链路懒加载体系：目录渲染、缩略图生成、文件预览、文件编辑均采用增量加载策略，杜绝全量一次性加载。
10. 与终端模块深度联动：当前目录一键唤起终端标签、文件路径拖拽至终端自动填充、终端路径点击跳转至文件管理器。
11. **关闭即销毁规则**：用户主动点击标签关闭按钮（X），立即移除该浏览会话；存在未保存编辑、进行中后台任务时触发二次确认，确认后关闭标签，后台任务可选择是否继续执行。

### 2.3 横向顶部标签栏交互
1. 标签从左到右横向排列，单标签由目录图标、目录名称、状态标记、关闭按钮组成。
2. 激活态标签通过背景样式区分，与主浏览面板视觉衔接；固定标签前置排列，附带固定标识。
3. **关闭按钮行为**：点击关闭按钮（X），无未保存编辑、无进行中关联任务时，直接移除标签、释放视图资源；存在未保存文件则弹出确认弹窗，存在进行中的下载/压缩任务则提示"关闭标签后任务是否继续"，用户选择后执行对应操作并关闭标签。
4. 标签栏最右侧固定新建标签按钮，点击创建新标签并打开默认起始目录（可配置为首页、根目录或上次浏览位置）。
5. 点击非激活标签，立即切换至对应目录会话，地址栏与侧边导航同步更新；当前标签的预览弹窗同步关闭，编辑状态后台保留。
6. 鼠标中键点击标签空白区域可快速新建标签；拖拽标签移出窗口可生成独立文件管理器窗口，拖回窗口则合并为标签。
7. 支持拖拽外部文件/文件夹至标签栏空白处，自动新建标签并打开对应目录。

### 2.4 竖向侧边标签栏交互

#### 2.4.1 形态切换控件
位于竖向标签栏顶部，点击可在压缩形态与展开形态间平滑切换，切换过程中标签顺序、激活状态、固定状态、编辑与预览会话均保持不变。

#### 2.4.2 展开形态
1. 标签纵向依次排列，单标签由「目录图标 + 目录名称文本 + 状态标记 + 关闭按钮」横向组合构成。
2. 激活态标签通过背景样式区分，关闭按钮常驻显示；存在后台任务或未保存编辑时，关闭标签触发二次确认。
3. 点击非激活标签，立即切换至对应目录会话，主面板与地址栏同步更新。
4. 固定标签置顶排列，附带固定标识，默认隐藏关闭按钮。
5. 标签名称过长自动截断，悬停展示完整绝对路径与当前目录统计信息。
6. 标签总数超出栏高时，支持鼠标滚轮纵向滚动浏览。
7. 栏底固定新建标签按钮，点击创建新标签并打开默认起始目录。
8. 支持拖拽文件/文件夹至标签区域，悬停延时后自动切换至对应目录。

#### 2.4.3 压缩形态
1. 标签纵向等距排列，仅展示目录图标，标签栏收窄至适配图标宽度，最大化文件浏览区域。
2. 悬停交互规则：
   - 悬停**当前激活态标签**：图标替换为关闭（X）图标，点击即触发关闭逻辑：无未保存内容与进行中任务则直接关闭标签；存在未保存内容或后台任务时弹出二次确认，确认后执行关闭。
   - 悬停**非激活态标签**：图标保持原样不变，点击则切换至对应目录会话。
3. 图标角标承载状态标识：加载中动画、访问错误标记、固定标签标识、后台任务进度提示、未保存编辑标记、预览中标记。
4. 支持拖拽文件/文件夹至图标标签，悬停延时后自动切换至对应目录，释放完成操作。
5. 标签总数超出栏高时，支持鼠标滚轮纵向滚动浏览。
6. 栏底固定新建标签按钮，点击创建新标签。

### 2.5 目录浏览懒加载机制
1. **虚拟滚动渲染**：单目录文件条目超过阈值（默认200条）时自动启用虚拟滚动，仅渲染可视区域内的文件条目，滚动时动态回收离屏条目、渲染新入屏条目；始终维持低量级DOM节点，避免海量文件目录下界面卡死。
2. **缩略图懒加载**：图标/缩略图视图模式下，所有文件先展示对应类型的默认图标作为占位；当条目滚动进入可视区域后，再异步生成/加载缩略图；加载失败或超时时自动回退为默认图标，不阻塞目录滚动。
3. **增量分页加载**：超量文件目录（万级以上条目）采用分批增量加载，首屏仅加载指定数量条目，滚动至列表底部时自动追加加载下一批；加载过程展示底部加载状态，全程可中断、可滚动，不阻塞界面交互。
4. **元信息异步补全**：目录渲染优先展示文件名与文件类型，文件大小、修改时间、权限、详情属性等元数据采用异步方式后台补全，不阻塞目录首屏呈现；网络目录与挂载目录优先加载缓存信息，后台同步更新最新数据。
5. **子目录延迟解析**：侧边导航栏与目录树仅默认加载首层目录，点击展开文件夹时再异步解析子级目录结构；深层级目录不一次性递归加载，避免打开大容量磁盘时界面无响应。

### 2.6 文件预览交互规范

#### 2.6.1 通用预览规则
1. **触发方式**：选中文件按空格键触发快速预览、双击文件（可配置为打开/预览/编辑）、右键菜单选择「预览」、地址栏直接输入文件路径自动进入预览。
2. **弹窗形态**：模态居中弹窗，从属于当前激活标签；切换标签时预览弹窗自动关闭，标签关闭时对应预览同步终止。
3. **弹窗通用结构**：
   - 顶部标题栏：显示文件名、文件大小、修改时间，右侧依次为「编辑」「下载」「在新标签打开」「用默认程序打开」「最大化」「关闭」按钮。
   - 中间主体区：自适应展示对应类型的预览内容。
   - 底部操作栏：根据文件类型展示对应操作控件，通用包含「上一个文件」「下一个文件」切换按钮。
4. **同目录切换**：同目录下可通过左右箭头按钮或快捷键切换上一个/下一个文件，自动加载对应预览内容，跳过不可预览格式。
5. **尺寸与拖拽**：支持拖拽弹窗边缘调整大小，支持最大化全屏预览；关闭后记忆窗口尺寸，下次打开沿用。
6. **加载与异常处理**：
   - 大文件、远程网络文件显示加载进度条与预估时长。
   - 文件损坏、格式不支持、权限不足时，展示对应错误提示与「重试」「选择打开方式」备选操作。
   - 超大文件超过阈值时，弹出确认提示，避免占用过多资源。
7. **懒加载总则**：所有类型文件预览均采用按需加载策略，不一次性读取全量文件内容；根据可视区域、播放进度、浏览位置动态加载对应片段。
8. **资源自动回收**：关闭预览弹窗后立即释放文件读取句柄与渲染缓存；关闭标签时，同步终止该标签下所有未完成的加载任务、清理预览缓存。

#### 2.6.2 分类型预览规则

##### 文本与代码文件
- 左侧常驻显示行号，支持一键切换显示/隐藏行号。
- 根据文件后缀自动匹配对应编程语言语法高亮，支持手动切换语言类型。
- 支持字体缩放、自动换行/强制不换行切换、文件编码格式识别与切换。
- 内置搜索框，支持关键词高亮查找、跳转到指定行号。
- 支持选中并复制文本内容，只读模式不可直接编辑。
- 分片懒加载：默认仅加载文件首部指定行数（默认500行），滚动至接近可视区域底部时自动追加加载下一分片；行号随分片同步生成，不预先计算全量行号。
- 超大文件保护：单文件超过体积阈值时，默认仅预览首部内容，支持手动选择加载更多或跳转至指定行；禁止全量加载超阈值大文件。

##### 文档类文件（PDF、Office 系列）
- PDF 文件：支持单页/双页浏览、滚动翻页、页码跳转、缩放适配、缩略图侧边栏、全文关键词搜索。
- Word/Excel/PPT 只读渲染：Word 支持翻页缩放；Excel 支持工作表切换、行列滚动；PPT 支持幻灯片逐页切换。
- 通用提供打印、另存为本地文件操作入口。
- 分页懒渲染：默认仅渲染当前可视页面，滚动临近时预加载前后2页，不一次性解析渲染全部页面；大页数文档采用分页异步渲染，每页独立加载。
- 缩略图侧边栏懒加载：文档缩略图仅生成可视区域内的页码缩略图，滚动时动态追加。

##### 图片类文件
- 自适应弹窗尺寸居中展示，支持鼠标滚轮缩放、拖拽平移查看细节。
- 支持顺时针/逆时针 90° 旋转、水平/垂直翻转。
- 底部信息栏显示图片分辨率、格式、文件大小、色彩模式。
- 同目录图片支持左右快捷切换，支持幻灯片自动播放模式。
- 渐进式加载：打开预览后先加载低分辨率缩略图快速占位，再异步加载高清原图。
- 超大图分块渲染：超高分辨率图片采用分块加载策略，仅渲染当前可视区域的图片区块，平移缩放时动态加载对应区块内容。
- 同目录预加载：切换上一张/下一张时，仅预加载相邻1张图片的缩略图，不批量预加载全目录图片。

##### 音视频媒体文件
- 内置播放器，打开后自动加载并开始播放，支持暂停/继续。
- 可拖拽进度条跳转播放位置，实时显示当前播放时长与总时长。
- 支持音量调节、静音切换、0.5x-2x 倍速播放。
- 视频支持全屏播放，全屏状态保留极简播放控件；支持画面比例自适应切换。
- 音频文件展示封面与播放波形，支持后台播放（可配置），切换标签时音频可继续播放。
- 支持本地字幕文件自动匹配加载。
- 流式分片加载：采用边播边加载的流式策略，仅缓冲当前播放点前后的指定时长数据，不预加载全量文件。

##### 压缩包文件
- 以目录树形式预览压缩包内部文件结构，展示文件总数、压缩率、原始大小与压缩后大小。
- 支持查看内部文件的文件名、大小、修改时间等基础信息。
- 压缩包内的文本、图片等可预览文件，支持直接双击嵌套预览。
- 提供「提取单个文件」「解压到当前目录」「解压到指定位置」操作入口。
- 分级懒解析：默认仅解析压缩包首层目录结构，点击展开内部文件夹时，再异步解析对应子级的文件列表；不一次性递归解析全压缩包结构。

##### 不支持格式
- 展示文件图标、基础属性信息，提示「该格式暂不支持预览」。
- 提供「用系统默认程序打开」「选择其他打开方式」操作按钮。

#### 2.6.3 预览快捷键
- 空格：触发选中文件预览 / 暂停音视频播放
- Esc：关闭预览弹窗
- ← / →：切换上一个 / 下一个文件
- Ctrl + 加号/减号：缩放预览内容
- F11：切换全屏预览
- Ctrl + F：预览内搜索

### 2.7 文件编辑交互规范

#### 2.7.1 触发入口
1. 文件右键菜单「编辑」选项，点击直接打开编辑模式。
2. 预览弹窗顶部「编辑」按钮，点击从只读预览模式切换为编辑模式，保留当前滚动位置与缩放配置。
3. 全局可配置双击文件默认行为：预览、直接编辑、用系统程序打开。
4. 地址栏输入文件完整路径，可配置默认进入预览或编辑模式。
5. 终端输出中的文件路径，支持右键选择「编辑文件」直接唤起编辑会话。

#### 2.7.2 编辑器核心能力
1. 左侧常驻行号，支持显示/隐藏切换；自动匹配文件后缀启用对应语法高亮，支持手动切换编程语言。
2. 基础编辑能力：撤销/重做、多光标编辑、选中复制粘贴、全选、行内/块级注释、行删除/复制/上下移动。
3. 搜索替换面板：支持关键词高亮、正则匹配、大小写区分、全部替换、逐个替换。
4. 视图配置：字体大小缩放、自动换行切换、制表符宽度设置、空格/制表符切换、文件编码识别与切换。
5. 支持跳转到指定行、跳转到文件首尾、快速定位到匹配的括号。

#### 2.7.3 编辑与标签联动
1. 文件进入编辑状态后，对应标签名称后追加 `*` 标记，标识存在未保存修改；竖向压缩形态通过角标标识未保存状态。
2. 切换标签时，编辑内容、光标位置、滚动位置完整保留，不丢失未保存修改；非激活标签的编辑会话后台驻留，不重复占用渲染资源。
3. 同一文件被多个标签同时打开编辑时，修改内容实时同步；保存时若检测到多标签冲突，弹出提示选择保留版本。
4. 编辑状态下关闭预览弹窗，编辑会话不终止，自动转为标签内常驻编辑视图。

#### 2.7.4 保存与异常处理
1. 支持 Ctrl+S 手动保存；可配置自动保存间隔，定时静默保存修改内容。
2. 保存时校验文件写入权限，权限不足、文件被占用、磁盘空间不足时，编辑器底部标红提示具体原因，保留全部修改内容不丢失。
3. 文件在外部被修改时，弹出提示框，提供三个选项：加载外部最新版本、保留本地修改、对比差异。
4. 关闭标签、关闭程序时，检测到未保存修改自动触发二次确认弹窗，列出所有未保存文件，防止内容丢失。

#### 2.7.5 大文件编辑保护
1. 超过体积阈值的文件默认禁止全量编辑，弹出风险提示，默认仅以只读预览模式打开。
2. 确认编辑大文件时，采用分片加载编辑机制，不一次性读取全量内容到内存；滚动到对应区域再加载对应分片内容，避免内存溢出。
3. 二进制文件、可执行文件默认禁止编辑，仅提供预览与属性查看，防止误修改导致文件损坏。

### 2.8 文件下载与压缩打包交互规范

#### 2.8.1 触发入口
1. **行内操作列**：单文件行尾常驻「下载」按钮；文件夹行尾常驻「打包下载」按钮，点击直接唤起压缩配置弹窗。
2. **右键菜单**：文件右键包含「下载」；文件夹右键包含「打包下载」「压缩打包」；多文件选中后右键包含「压缩打包」。
3. **顶部工具栏**：选中单个/多个文件与文件夹后，工具栏出现「下载」「压缩」操作按钮。
4. **预览/编辑弹窗**：所有可预览、可编辑文件的弹窗顶部，均提供「下载」快捷入口。

#### 2.8.2 单文件下载逻辑
1. 点击下载后直接触发下载，中小文件后台静默执行，大文件实时展示下载进度。
2. 按文件体积自适应下载策略：小文件直接流式下载；大文件采用分片流式下载，不占用前端内存；超过体积阈值的超大文件触发二次确认，避免误操作。
3. 下载过程不阻塞目录浏览与标签切换，可在全局后台任务面板查看进度、暂停与取消。
4. 关闭对应标签时，提示是否终止关联的下载任务；选择继续则任务转入全局后台执行，选择终止则立即取消并清理临时文件。
5. 下载完成后给出系统通知提示；下载失败展示错误原因与重试入口。

#### 2.8.3 压缩配置弹窗
触发场景：文件夹点击打包下载、多文件选中点击压缩、右键选择「压缩打包」时弹出模态窗口。
1. **压缩类型**：下拉选择控件，默认选中 `tar.gz (推荐)`，可选 zip、tar.bz2 等通用压缩格式；不同格式附带轻量提示，标注适用场景与兼容性。
2. **压缩路径**：输入框默认填充当前所在目录，自动生成带随机标识的压缩包文件名（规则：原名称_随机字符串.对应后缀）；支持手动修改文件名与保存的目标路径。
3. **路径选择按钮**：输入框右侧文件夹图标，点击弹出目录选择器，可可视化指定压缩包的保存位置。
4. **压缩并下载**：复选框选项，勾选后压缩完成自动触发压缩包下载；未勾选则仅在指定路径生成压缩包文件，不自动下载。
5. **操作按钮**：
   - 取消：关闭弹窗，不执行任何操作，不保留已修改的配置。
   - 压缩：先校验路径写入权限、磁盘剩余空间、文件名字法性；校验通过则关闭弹窗，启动后台压缩任务。
6. **异常校验**：目标路径无写入权限、磁盘空间不足、文件名含非法字符时，对应输入项标红并展示错误提示，禁止提交执行。

#### 2.8.4 压缩任务后台联动
1. 压缩任务全程后端独立执行，弹窗关闭、关闭对应标签均不中断任务运行；关闭标签时可选择是否继续保留任务。
2. 所属标签展示进度角标，全局后台任务面板可查看实时进度、已处理大小、预估剩余时间。
3. 支持中途取消压缩任务，取消后自动清理已生成的临时文件与不完整压缩包。
4. 压缩完成后自动刷新对应目录，显示生成的压缩包；勾选「压缩并下载」则自动触发压缩包下载。
5. 压缩失败时弹出错误提示，说明失败原因，提供重试入口。
6. 大容量文件夹压缩采用增量遍历统计，不一次性全量扫描文件，避免界面卡顿。

### 2.10 标签状态规则
- **正常状态**：显示默认目录图标与名称，可正常浏览与操作。
- **加载中**：目录内容加载时，标签展示加载动画，可切换标签，加载任务后台挂起。
- **路径无效**：目录不存在或已删除时，图标置灰并附加错误标识，点击标签提示路径失效，支持重试或一键返回上级目录。
- **权限不足**：无访问权限的目录，标签增加权限标识，主面板同步提示访问受限。
- **未保存编辑**：标签内存在未保存的文件修改时，名称后追加 `*` 标记，关闭标签时触发二次确认；持久化缓存后重启仍保留未保存状态。
- **后台任务进行中**：标签展示进度角标，提示存在文件传输、压缩解压、下载等后台操作；点击角标可打开任务面板查看详情与管理；关闭标签时可选择是否保留后台任务。
- **预览中**：当前标签存在打开的预览弹窗时，标签增加预览中标识；音视频后台播放时标签显示播放状态角标。
- **网络目录离线**：网络共享/挂载目录离线时，标签增加离线标识，目录内容置灰，仅支持查看本地缓存内容。

### 2.12 资源与性能管控
1. **渲染降级**：非激活标签暂停目录视图渲染与缩略图加载，仅保留路径与状态数据；切换标签时优先加载可视区域内容。
2. **缓存分级**：目录元数据、缩略图、预览内容分级缓存，定期清理长时间未访问标签的缓存资源。
3. **编辑资源回收**：长时间未激活的编辑标签自动释放渲染资源，仅保留修改内容内存快照，切回时快速恢复。
4. **后台任务限流**：多标签同时存在下载、压缩等后台任务时，自动限流排队，避免同时满载导致界面卡顿。
5. **异常隔离**：单标签目录加载崩溃、预览异常，不影响其他标签正常运行，异常标签支持一键刷新恢复。

### 2.13 标签与会话持久化机制
1. **标签全状态持久化**：所有文件管理器标签的完整状态实时持久化落盘，包括：标签排序、固定状态、当前目录路径、视图模式、排序筛选规则、浏览历史栈、文件选中状态。关闭WebUI、程序异常重启后，按原样完整还原标签栏。**用户主动关闭的标签，从持久化列表中移除，重启不自动恢复**。
2. **编辑会话持久化**：
   - 编辑中的文件内容，采用定时自动落盘+变更即时缓存双机制，意外退出、程序崩溃不会丢失未保存修改。
   - 程序重启后，自动还原所有异常退出前的编辑标签，标记未保存状态，恢复光标位置、滚动位置与编辑历史，支持继续编辑或撤销。
   - 缓存文件与原文件绑定校验，原文件已被删除时，提示缓存文件信息，支持另存为恢复。
   - 用户主动关闭带未保存内容的标签并确认放弃修改时，立即清理对应编辑缓存。
3. **后台任务持久化托管**：下载、压缩、解压等后台任务，由后端独立进程托管，关闭WebUI页面、关闭对应标签（选择保留任务时）均不中断任务执行。主程序重启后，自动扫描接管仍在运行的后台任务，同步到全局任务面板。
4. **启动恢复策略**：
   - 程序启动时，优先还原异常退出前的标签栏结构与状态，目录内容采用懒加载：激活标签立即加载目录内容，非激活标签仅保留路径信息，切换时再加载。
   - 可配置启动恢复规则：恢复上次全部标签、仅恢复固定标签、不恢复、仅恢复最近N个标签。
   - 用户正常主动关闭的标签，不纳入自动恢复范围。
5. **浏览记忆持久化**：每个目录的视图配置、排序方式、筛选条件按路径持久化记忆，下次打开该目录时自动沿用；全局统一生效，不随标签销毁而丢失。
6. **异常兜底**：程序异常崩溃前，自动触发紧急落盘，写入所有标签、编辑、任务的快照；下次启动时检测到异常退出，自动提示恢复，并列出可恢复的会话列表供用户选择。

## 三、模块联动规则
1. **目录联动终端**：文件管理器任意目录下，通过右键菜单、工具栏「在终端中打开」，自动新建终端标签并直接跳转至对应工作目录；终端会话独立保活，不受文件标签关闭影响。
2. **路径双向跳转**：终端输出中的文件/目录路径支持点击，自动新建/切换至文件管理器标签并定位到对应位置；文件拖拽至终端标签，自动在终端命令行中填入文件绝对路径。
3. **编辑终端联动**：终端内调用编辑器指令，可唤起文件管理器的编辑会话，修改保存后自动同步至终端环境。
4. **状态统一管理**：全局后台任务面板统一展示终端、文件管理器的所有后台任务，支持集中暂停、取消与进度查看；所有任务均持久化托管，不受前端关闭影响。
5. **布局全局同步**：标签栏布局（横向/竖向、展开/压缩）全局生效，终端与文件管理器标签同步切换，保持界面一致性；布局偏好持久化存储，重启后沿用。
